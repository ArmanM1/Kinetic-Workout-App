## 2024-05-24 - Mobile Keyboard Optimization for Search Inputs\n**Learning:** Next.js input components without type='search' or explicit aria-labels lack semantic meaning for screen readers and default to generic soft keyboards on mobile (especially iOS), leading to a suboptimal mobile experience.\n**Action:** Always add type='search' and aria-label='Search...' to global and local search inputs to trigger native mobile search keyboards and ensure accessible context.
